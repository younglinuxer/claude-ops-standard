# 运维安全总则

本文件适用于所有开发环境和测试环境的运维操作。Claude 必须先读本文件，再读取具体场景标准和项目适配清单。

## 操作分级

### 可直接执行的只读操作

- 查看目录、配置文件、脚本和文档。
- 查看服务状态、容器状态、端口占用。
- 查看日志，但不能输出密钥、Token、密码、证书内容。
- 查看 Git 状态和差异。

### 需要命令级确认的变更操作

- 启动、停止、重启服务或容器。
- 修改 Nacos、环境变量、Compose 文件、启动脚本。
- 执行数据库 DDL 或 DML。
- 发布、回滚、替换镜像或 jar 包。
- 清理日志、缓存、临时文件、Docker volume、镜像、容器。

### 默认禁止的危险操作

除非用户明确要求、确认目标环境和回滚方案，否则不得执行：

- `rm -rf`、批量删除、递归移动关键目录。
- `docker compose down -v`、`docker volume rm`、`docker system prune`。
- `DROP`、`TRUNCATE`、无 `WHERE` 的 `DELETE` 或 `UPDATE`。
- 直接修改生产环境。
- K8s 写操作，例如 `apply`、`delete`、`scale`、`rollout restart`、`edit`、`patch`。
- 输出或新增明文密码、Token、私钥、证书。

## 命令级确认格式

执行变更命令前，Claude 必须先给出以下内容：

```text
目标环境：
目标主机/目录：
操作目的：
将执行的命令：
影响范围：
验证方式：
回滚方式：
需要用户确认：
```

用户确认后，按列出的命令执行。执行中如发现目标环境、路径、服务名、数据库名与计划不一致，必须停止并重新确认。

## 环境确认规则

- 任何变更前必须确认当前目录、目标环境、目标服务。
- 开发和测试环境名称不得混用；不清楚时先查项目适配清单。
- 不得根据记忆猜测服务器路径、Compose 文件位置、Nacos namespace 或数据库名。
- 文档缺失时，先补充事实或向负责人确认，不得自行编造。
