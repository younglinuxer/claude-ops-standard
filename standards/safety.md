# 运维安全总则

本文件适用于所有开发环境和测试环境的运维操作。Claude 必须先读本文件，再读取具体场景标准和项目适配清单。

## 操作分级

### 可直接执行的只读操作

- 查看目录、配置文件、脚本和文档。
- 查看服务状态、容器状态、端口占用。
- 查看日志，但不能输出密钥、Token、密码、证书内容。
- 查看 Git 状态和差异。

### 需要命令级确认的变更操作

- 启动、停止、重启服务或容器。
- 修改 Nacos、环境变量、Compose 文件、启动脚本。
- 执行数据库 DDL 或 DML。
- 发布、回滚、替换镜像或 jar 包。
- 安装或卸载系统组件。
- 修改端口、防火墙、系统服务、Docker 配置或数据目录。
- 清理日志、缓存、临时文件、Docker volume、镜像、容器。

### 默认禁止的危险操作

除非用户明确要求、确认目标环境和回滚方案，否则不得执行：

- `rm -rf`、批量删除、递归移动关键目录。
- `docker run` 启动长期服务，或用 `docker start/restart/rm` 手工管理运行服务。
- `docker compose down -v`、`docker volume rm`、`docker system prune`。
- `DROP`、`TRUNCATE`、无 `WHERE` 的 `DELETE` 或 `UPDATE`。
- 直接修改生产环境。
- K8s 写操作，例如 `apply`、`delete`、`scale`、`rollout restart`、`edit`、`patch`。
- 输出或新增明文密码、Token、私钥、证书。

## 命令级确认格式

执行变更命令前，Claude 必须先给出以下内容：

```text
目标环境：
目标主机/目录：
操作目的：
将执行的命令：
影响范围：
验证方式：
回滚方式：
是否属于服务器重大变更：
重大变更日志文件：/data/logs/{project}-{env}-changes.md
需要用户确认：
```

用户确认后，按列出的命令执行。执行中如发现目标环境、路径、服务名、数据库名与计划不一致，必须停止并重新确认。

## 服务器重大变更记录

以下操作属于服务器重大变更，执行后必须追加记录到 `/data/logs/{project}-{env}-changes.md`：

- 安装或卸载系统组件。
- 修改 Docker、Compose、端口、防火墙、系统服务或数据目录。
- 升级、回滚、替换镜像或 jar 包。
- 变更 Nacos、数据库连接或中间件持久化路径。
- 清理数据卷、容器、镜像、宿主机数据或关键日志。

记录内容必须包含：

```text
时间：
操作人：
环境：
主机：
变更原因：
执行命令：
影响范围：
备份位置：
回滚方式：
验证结果：
```

## 环境确认规则

- 任何变更前必须确认当前目录、目标环境、目标服务。
- 开发和测试环境名称不得混用；不清楚时先查项目适配清单。
- 不得根据记忆猜测服务器路径、Compose 文件位置、Nacos namespace 或数据库名。
- Docker 运行必须使用项目登记的单一 Compose 文件。
- 文档缺失时，先补充事实或向负责人确认，不得自行编造。
