# 数据库操作标准

本标准适用于开发环境和测试环境。生产数据库不在第一版授权范围内。

## 只读优先

排查问题时优先执行：

```sql
SELECT ...
SHOW TABLES;
SHOW CREATE TABLE table_name;
EXPLAIN SELECT ...
```

查询输出不得包含密码、Token、私钥、证件号等敏感数据；如必须查看，先说明原因并限制字段和行数。

## 强确认操作

以下操作必须命令级确认：

- `CREATE`、`ALTER`、`DROP`。
- `INSERT`、`UPDATE`、`DELETE`。
- 执行 SQL 脚本。
- 导入、导出、恢复备份。

确认内容必须包含：

```text
目标环境：
数据库地址来源：
数据库名：
影响表：
SQL 或脚本路径：
预计影响行数：
备份方式：
回滚方式：
验证 SQL：
```

## 危险 SQL 规则

- `DROP`、`TRUNCATE` 默认禁止，除非用户明确确认。
- 无 `WHERE` 的 `DELETE` 或 `UPDATE` 默认禁止。
- `WHERE 1=1`、空条件、拼接条件不明确，按无 `WHERE` 处理。
- 批量更新前必须先执行等价 `SELECT COUNT(*)` 估算影响行数。
- DDL 前必须确认当前库名，禁止依赖默认连接库。

## 备份要求

- 结构变更前至少备份表结构。
- 数据变更前根据影响范围备份目标表或目标行。
- 备份文件路径、时间、库名、表名必须记录在执行结果中。
- 备份命令不得把密码写入仓库文件。

## 敏感信息

- 文档中只写连接信息来源，不新增明文密码。
- 命令历史中尽量避免出现密码；优先使用客户端配置、环境变量或临时交互输入。
- SQL 文件不得包含真实密码、Token、私钥。
