# FAQ：开发/测试环境常见问题

## Claude 可以直接重启服务吗？

不可以直接执行变更命令。必须先读取通用标准和项目适配清单，输出命令级计划，用户确认后再执行。

## 找不到 Docker Compose 文件怎么办？

先读项目适配清单。如果适配清单没有登记真实位置，不得猜路径或搜索后直接操作；应要求补充 Compose 位置、环境和负责人。

## 可以直接改数据库吗？

开发和测试环境允许在强确认后执行。确认内容必须包含环境、库名、影响表、SQL、预计影响行数、备份方式和回滚方式。

## 可以执行无 WHERE 的 UPDATE 或 DELETE 吗？

默认禁止。除非用户明确确认目标环境、影响范围和回滚方案，并且该操作确实符合业务目的。

## K8s 环境可以怎么处理？

第一版只允许只读排查，例如 `get`、`describe`、`logs`、`top`。任何 `apply/delete/scale/rollout restart/edit/patch` 都必须升级给负责人确认。

## 密码和 Token 写在哪里？

不写入标准文档或项目适配清单。文档只记录配置来源，例如 Nacos、服务器私有配置或负责人指定渠道。

## 前端 `.env.*` 是 API 地址事实源吗？

不一定。多数 Vue 项目还会有运行时配置文件。必须读取项目适配清单确认，例如 v2x 的 API/WS 事实源是 `frontend/cv2x-monitor-web/public/config.js`。
