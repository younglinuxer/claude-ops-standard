# SOP：发布与回滚

## 发布前

- 确认目标环境、服务名、版本号。
- 确认 Docker 镜像地址，必须以 `registry.cn-sccd1.ctyun.cn/cmat/` 开头。
- 确认镜像 tag、目标 Compose 服务名和当前回滚镜像 tag。
- 开发推送镜像，目标环境由运维或负责人替换 Compose 中的镜像 tag。
- 记录当前镜像地址、镜像 tag、镜像构建来源 jar 文件名或 Git commit。
- 确认打包配置和运行配置来源，配置必须可外部修改，不得写死在代码中。
- 确认配置是否需要同步变更。
- 输出命令级计划并等待确认。

## 发布后

- 查看服务状态。
- 查看启动日志。
- 验证健康检查和关键接口。
- 记录实际版本、镜像地址、镜像 tag、配置来源和验证结果。

## 回滚

- 使用发布前记录的镜像地址、镜像 tag 或 Git commit 回滚。
- 回滚后重复发布后验证。
- 回滚失败时停止继续操作，升级给负责人。
